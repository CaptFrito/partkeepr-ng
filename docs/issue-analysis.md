# Upstream issue analysis (weighted ranking)
_Generated by `scripts/analyze-issues.py`. **This is input for review, not adopted scope.** The user wants to evaluate each candidate before treating it as a requirement._

## Method

Issues are scored by an explicit weighted sum. PRs are excluded. Higher score ≈ stronger signal that this issue represents real, current user pain.

Score components (tunable in `scripts/analyze-issues.py`):
- `comment_per` = 1.0
- `comment_cap` = 20
- `reaction_per` = 0.5
- `reaction_cap` = 10
- `state_open_bonus` = 5.0
- `recent_close_bonus` = 1.0
- `label_bug` = 2.0
- `label_feature` = 1.0
- `label_default` = 0.5
- `feature_match_bonus` = 1.5
- `feature_match_cap_n` = 3

_Recent-close window: 4 years._
